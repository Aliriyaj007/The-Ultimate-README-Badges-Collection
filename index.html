Mastering Custom Badges

Everything you need to know about creating, styling, and customizing badges for your README.

1 The Anatomy of a Badge

Most badges are generated using Shields.io. The basic URL structure is simple:

https://img.shields.io/badge/<LABEL>-<MESSAGE>-<COLOR>

  • LABEL: The left side text (e.g., "build").
  • MESSAGE: The right side text (e.g., "passing").
  • COLOR: The background color of the message (e.g., "green", "ff0000").

Example:

![Status](https://img.shields.io/badge/Status-Active-green)

2 Choosing a Style

You can change the look of any badge by adding the ?style= parameter.

Style Name Code Preview
Flat (Default) flat
Flat Square flat-square
Plastic plastic
For The Badge for-the-badge
Social social

3 Adding Icons & Logos

Make your badges recognizable by adding brand logos from Simple Icons using the ?logo= parameter.

Steps:

  1. Find a slug from Simple Icons (e.g., react, python, gmail).
  2. Add it to the URL: ?logo=react.
  3. (Optional) Change logo color: ?logoColor=white.

Examples:

?logo=react&logoColor=61DAFB
?logo=python&logoColor=white
?logo=gmail&logoColor=white 
![React](https://img.shields.io/badge/React-20232a?logo=react&logoColor=61DAFB)

4 Dynamic Badges (Advanced)

You can create badges that update automatically by pulling JSON data from any API.

Endpoint: https://img.shields.io/badge/dynamic/json

Required Parameters:

  • url: The API URL to fetch.
  • query: JSON path to the specific data field (e.g., $.key).
  • label: The label text.

Example (Package Version):

https://img.shields.io/badge/dynamic/json?url=YOUR_JSON_URL&query=$.version&label=Version

üí° Pro Tips

  • Spaces: Use %20 or _ for spaces (e.g., Hello%20World).
  • Colors: You can use Hex codes (without the #) like 2563eb or named colors like blue, brightgreen, critical.
  • Cache: Badges are cached. If you make a quick change and it doesn't update, wait a few minutes or add a random query param like ?cache=1.
