## ‚ö° Quick Start: Add a Badge in 2 Minutes

1.  **Find the right category** in the `/Categories/` folder.
2.  **Copy the format** of existing badges in that file.
3.  **Add your badge** to the table.
4.  **Open a Pull Request**.

That's it! We usually merge valid PRs within 24-48 hours.

## üõë Contribution Rules

To ensure high quality, please follow these simple rules:

### Do ‚úÖ
*   **Use reliable providers:** Prefer [Shields.io](https://shields.io/), official service badges, or well-maintained generators.
*   **Check visual consistency:** If adding to a list of "Classic" badges, don't add a "For-the-badge" style unless there is a specific section for it.
*   **Verify the link:** Ensure the badge renders and the link destination works.
*   **Alphabetical Order:** Try to keep lists alphabetized if possible, it helps everyone find things.

### Don't ‚ùå
*   **No broken images:** If the badge doesn't load, we can't merge it.
*   **No duplicates:** Check if the badge already exists in the file (or another category).
*   **No "Novelty" without utility:** Purely decorative formatting quirks that aren't reusable badges should go in `10-custom-other.md`.

## üõ†Ô∏è How to Submit a Pull Request

1.  **Fork** the repository on GitHub.
2.  **Clone** your fork locally:
    ```bash
    git clone https://github.com/your-username/The-Ultimate-README-Badges-Collection.git
    ```
3.  **Create a branch** for your changes:
    ```bash
    git checkout -b add-new-badge
    ```
4.  **Edit the file** (e.g., `Categories/01-build-ci-status.md`) and add your badge markdown.
5.  **Commit** your changes:
    ```bash
    git commit -m "Add [Name] badge to Build Status"
    ```
6.  **Push** to your fork:
    ```bash
    git push origin add-new-badge
    ```
7.  **Submit a Pull Request** on the main repository.
